Performance Optimization

메시지 조회 성능 개선

Problem

채팅 메시지가 증가하면서 이전 메시지 조회 API의 응답 속도가 점점 느려졌습니다.

Analysis

EXPLAIN 실행 계획을 분석한 결과 불필요한 레코드 탐색이 발생하고 있었습니다.

Solution

chat_room_id + id 복합 인덱스를 적용하고 Cursor Pagination으로 조회 방식을 개선했습니다.

Before & After

Before After
조회 레코드 395 rows 20 rows
응답 시간 10.4 ~ 15.1 ms 0.06 ~ 0.09 ms
실행 계획 분석과 복합 인덱스 적용을 통해 조회 성능을 크게 개선했으며, 기능 구현뿐 아니라 성능 최적화의 중요성을 경험했습니다.

Trouble Shooting

개발 과정에서 해결한 주요 문제

Issue 01

Redis PatternTopic을 사용하여 메시지를 구독하는 과정에서 실제 채널명이 아닌 패턴(chat:*)만 전달되는 문제가 발생했습니다.

Cause

PatternTopic의 콜백에서는 실제 Publish Channel이 아닌 Pattern 문자열이 전달되는 구조였습니다.

Solution

MessageListener를 직접 구현하여 실제 Channel 정보를 가져오도록 변경했고 채팅방별 메시지를 정상적으로 분기 처리했습니다.

Result

  • 멀티 서버 환경에서도 정상적인 메시지 전달
  • 채팅방별 Publish / Subscribe 분리
  • Redis Pub/Sub 구조 안정화
